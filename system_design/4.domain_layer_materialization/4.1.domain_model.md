## Design Principles

The domain layer is the semantic core of the system. It defines what exists, what state is valid, what transitions are allowed, and what must never happen. It contains no I/O, no frameworks, no LLM calls.

- **Aggregates** enforce their own invariants through guarded behavioral methods — illegal transitions raise domain errors.
- **Value objects** are immutable and self-validate on construction.
- **Domain events** are immutable facts — past tense, minimal payload.
- **Process entities** enforce state transition guards at all times.
- **External reference value objects** (User, School) are immutable pointers to entities managed by other bounded contexts. They are pre-resolved externally and passed in — never resolved or mutated inside this domain.
- **Immutable historical entities** (ConversationHistory) are created once, persisted, never updated. Append-only logs enforce immutability through lifecycle design.
- **Cross-context value objects** (RetrievedEvidence, RetrievalQuery) flow as function return values — no shared imports between domain layers.

## BC-1 — Conversation Management

### Aggregate Root: `ConversationSession`

> **Classification:** Process Entity  
> **Identity:** `session_id`  
> **Responsibility:** Lifecycle owner of a multi-turn user conversation. Manages session state transitions from creation to closure. Enforces turn sequencing and session-level invariants.

#### Properties

| Property      | Type            | Role               |
|---------------|-----------------|--------------------|
| `session_id`  | `str`           | identity (UUID)    |
| `user_id`     | `str`           | identity reference |
| `school_id`   | `str`           | identity reference |
| `status`      | `SessionStatus` | state              |
| `turn_count`  | `int`           | default: 0         |
| `created_at`  | `datetime`      |                    |
| `updated_at`  | `datetime`      |                    |

#### State Machine — `SessionStatus`

```
created → initializing → idle ⇄ active → closed
```

| From          | To            | Trigger                       | Emits          |
|---------------|---------------|-------------------------------|----------------|
| `created`     | `initializing`| `begin_initialization()`      | —              |
| `initializing`| `idle`        | `complete_initialization()`   | `SessionStarted` |
| `idle`        | `active`      | `receive_message()`           | —              |
| `active`      | `idle`        | `complete_turn(intent, response)` | `TurnProcessed` |
| `idle`        | `closed`      | `close()`                     | `SessionClosed` |

**Illegal transitions** (raise domain error):
- `created → active` — must initialize first
- `closed → any` — terminal state
- `active → closed` — must return to idle first
- `any → initializing` — only from `created`

#### Behavior

| Method | Guards | Side Effects |
|--------|--------|--------------|
| `begin_initialization()` | `status == created` | transitions to `initializing` |
| `complete_initialization()` | `status == initializing` | transitions to `idle`; emits `SessionStarted` |
| `receive_message()` | `status == idle` | transitions to `active` |
| `complete_turn(intent, response)` | `status == active` | increments `turn_count`; transitions to `idle`; emits `TurnProcessed` |
| `close()` | `status == idle` | transitions to `closed`; emits `SessionClosed` |

#### Factory

```
ConversationSession.create(user_id: str, school_id: str) → ConversationSession
```
Produces a session in `created` state with auto-generated `session_id` and `turn_count = 0`.

#### Invariants Enforced
- **INV-CS-1** — status must be one of the five valid states
- **INV-CS-2** — transition guards enforced at all times
- **INV-CS-6** — `user_id` must be provided (pre-resolved by external auth)

## BC-2 — Decision Intelligence

### Aggregate Root: `DecisionContext`

> **Classification:** Process Entity  
> **Identity:** `context_id` (equals `session_id` — 1:1)  
> **Responsibility:** The core intelligence asset. Progressively accumulates user preference signals across turns. Signals are never removed — accumulation is strictly monotonic. School anchor always present. State drives retrieval readiness.

#### Properties

| Property          | Type                    | Role                    | Notes                             |
|-------------------|-------------------------|-------------------------|-----------------------------------|
| `context_id`      | `str`                   | identity                | equals `session_id`               |
| `session_id`      | `str`                   |                         |                                   |
| `school_anchor`   | `SchoolAnchor`          | always present          | pre-seeded at session init; never changes |
| `signals`         | `list[PreferenceSignal]`| accumulating            | monotonic; never shrinks          |
| `identified_gaps` | `list[str]`             |                         | updated by gap evaluation         |
| `status`          | `ContextStatus`         | state                   |                                   |
| `created_at`      | `datetime`              |                         |                                   |
| `updated_at`      | `datetime`              |                         |                                   |

#### State Machine — `ContextStatus`

```
empty → partial ⇄ partial → ready → enriched ⇄ enriched
```

| From       | To         | Trigger                                             | Emits           |
|------------|------------|-----------------------------------------------------|-----------------|
| `empty`    | `partial`  | `add_signal()` — first non-school signal            | `ContextUpdated` |
| `partial`  | `partial`  | `add_signal()` — further signals, gaps remain       | `ContextUpdated` |
| `partial`  | `ready`    | `mark_ready()` — gap evaluation passed by app layer | `ContextReady`  |
| `ready`    | `enriched` | `add_signal()` — signal added after readiness       | `ContextUpdated` |
| `enriched` | `enriched` | `add_signal()` — further refinement                 | `ContextUpdated` |

**Illegal transitions** (raise domain error):
- `ready → partial / empty` — no regression
- `enriched → any lesser state` — no regression
- `empty → ready` — must pass through `partial`

#### Behavior

| Method | Guards | Side Effects |
|--------|--------|--------------|
| `add_signal(signal)` | signal not already present | appends; auto-transitions `empty→partial`, `ready→enriched`; emits `ContextUpdated` |
| `record_gap_evaluation(gaps)` | `status == partial` | updates `identified_gaps`; emits `GapDetected` if gaps non-empty |
| `mark_ready()` | `status == partial`, `len(signals) > 0` | transitions to `ready`; emits `ContextReady` |
| `is_ready_for_retrieval() → bool` | — | returns `True` if status is `ready` or `enriched` |
| `has_gap() → bool` | — | returns `True` if `status == partial` and `identified_gaps` non-empty |
| `snapshot() → dict` | — | returns read-only primitive snapshot; exposes no mutable internals |

#### Factory

```
DecisionContext.create(session_id: str, school_anchor: SchoolAnchor) → DecisionContext
```
Produces context in `empty` state. `school_anchor` must not be null (INV-DC-2).

#### Invariants Enforced
- **INV-DC-1** — status must be one of four valid states
- **INV-DC-2** — `school_anchor` always present; never null
- **INV-DC-3** — state never regresses; transition guards enforced
- **INV-DC-5** — `add_signal()` only appends; never removes
- **INV-DC-6** — `mark_ready()` requires at least one signal beyond school anchor

## Cross-Context Data Flows

All cross-context payloads travel as function return values — never as shared domain imports.

```
BC-1 app layer
  │  IDecisionIntelligenceService.process_turn(session_id, intent, message, turn_number)
  ▼
BC-2 app layer
  │  IKnowledgeRetrievalService.retrieve(RetrievalQuery)
  ▼
BC-3 app layer
  │  returns RetrievedEvidence (raw chunks, scores=0.0)
  ▲
BC-2 — IEvidenceEvaluator scores evidence (recall + precision)
  │  returns RetrievedEvidence (scored)
  ▲
BC-1 — assembles grounded AssistantResponse
```

## Invariant Enforcement Map

### Domain Layer (structural + state invariants)

| Invariant   | Enforced By |
|-------------|-------------|
| INV-CS-1    | `ConversationSession` state machine guard |
| INV-CS-2    | `ConversationSession` transition guards — illegal transitions raise domain error |
| INV-CS-6    | `ConversationSession.create()` — `user_id` must not be null |
| INV-DC-1    | `DecisionContext` state machine guard |
| INV-DC-2    | `DecisionContext.create()` — `school_anchor` must not be null |
| INV-DC-3    | `DecisionContext` transition guards — regression raises domain error |
| INV-DC-5    | `DecisionContext.add_signal()` — append-only; never removes |
| INV-DC-6    | `DecisionContext.mark_ready()` — guards `len(signals) > 0` |
| INV-RQ-2    | `RetrievalQuery.from_context()` factory — `source_target` derived from intent |
| INV-RQ-3    | `RetrievalQuery.from_context()` factory — validated against intent |
| INV-AR-5 *(construction)* | `AssistantResponse.create()` — validates score ranges |

### Application Layer (workflow + quality gate invariants)

| Invariant   | Enforced By |
|-------------|-------------|
| INV-CS-3    | `StartSessionUseCase` — school pre-seeded before first turn |
| INV-CS-4    | `ProcessTurnUseCase` — turn rejected if session not in `active` state |
| INV-CS-5    | `StartSessionUseCase` — checks for existing active/idle session before creating |
| INV-CS-7    | `StartSessionUseCase` — one active/idle session per user |
| INV-DC-4    | `DecisionIntelligenceSubgraph` — retrieval only activates when intent + context state both qualify |
| INV-RQ-1    | `DecisionIntelligenceSubgraph` — `query_text` constructed from context snapshot, not raw message |
| INV-AR-1    | `TurnGraph evaluate_response` — `groundedness_score` must reach threshold |
| INV-AR-2    | `IResponseGenerator` — LLM grounded on retrieved product evidence only |
| INV-AR-3    | `IResponseGenerator` — LLM grounded on retrieved referral rules only |
| INV-AR-4    | `IResponseGenerator` prompt — school reference injected into generation context |
| INV-AR-5    | `TurnGraph evaluate_response` — routes to fallback if scores fail |
| INV-AR-6    | `IResponseEvaluator` — `relevance_score` computed via reverse-question technique |
| INV-RE-1    | `IEvidenceEvaluator` — `context_recall_score` computed after retrieval |
| INV-RE-2    | `IEvidenceEvaluator` — `context_precision_score` computed after retrieval |
| INV-RE-3    | `DecisionIntelligenceSubgraph` retry edge — recall below threshold triggers query refinement |
| INV-RE-4    | `DecisionIntelligenceSubgraph` — evidence with failed scores does not reach generation |
| INV-WF-1    | `TurnGraph` — `classify_intent` executes before any other node |
| INV-WF-2    | `DecisionIntelligenceSubgraph` — context update before query construction |
| INV-WF-3    | `TurnGraph` — `process_decision_intelligence` before `generate_response` |
| INV-WF-4    | `TurnGraph` — no bypass path; decision intelligence executes on every turn |
| INV-WF-5    | `DecisionIntelligenceSubgraph` — `evaluate_evidence_quality` after retrieval, before returning evidence |
| INV-WF-6    | `TurnGraph` — `evaluate_response` after generation, before return |
