# System Workflows

# The System Has Only 3 Core Workflows

To keep it simple and strong, the system should only have:

1. Decision Context Building Workflow
2. Product Recommendation Workflow
3. Referral Guidance Workflow

That’s it.  
Everything else is internal mechanics.

---

# Workflow 1 — Decision Context Building

## Purpose
Progressively understand the user's needs and fundraising intent.

## Flow

User message  
→ extract signals  
→ update decision context  
→ determine missing info  
→ ask clarifying question (optional)  
→ continue conversation  

This directly comes from:

conversation → extract signals → update context → guide decisions

## Example

User:
"My kids are picky"

System:
stores preference

Assistant:
"Are you looking for snacks or breakfast?"

This workflow:

- builds context
- guides conversation
- prepares recommendation

This is the decision modeling workflow.

---

# Workflow 2 — Product Recommendation

This workflow activates when the system has enough context.

## Purpose
Guide user toward product decisions.

## Flow

Decision context available  
→ query product data  
→ match preferences  
→ generate recommendation  
→ explain reasoning  
→ continue conversation  

## Example

Context:

- picky kids
- healthy

Assistant:
"These healthy snack options work well for picky kids…"

This implements:

context-driven recommendation behavior.

---

# Workflow 3 — Referral Guidance

This workflow activates when:

- user asks about referrals
- user wants to support school more
- system detects opportunity

## Purpose
Guide user toward referral actions.

## Flow

User intent about fundraising  
→ retrieve referral rules  
→ explain mechanics  
→ suggest referral action  
→ continue conversation  

## Example

User:
"How can I help my school more?"

Assistant:
"Inviting other parents increases donations…"

This aligns with business requirement:
assistant explains referrals and encourages participation.

---

# How These Workflows Connect

They are not separate conversations.  
They are modes inside the same decision loop.

Conversation starts:

Workflow 1 — build context

Then:

Workflow 2 — recommend products

Then optionally:

Workflow 3 — guide referrals

All inside one conversation.
