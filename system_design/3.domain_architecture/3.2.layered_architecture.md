# Layered Architecture
## Carton Caps Conversational Assistant

---

## Architecture Pattern

**Clean Architecture** — dependencies always point inward.

```
infrastructure → application → domain
shell/api      → application (use cases only)
shell/composition wires everything — imports across all layers
```

Domain depends on nothing. Application depends only on domain. Infrastructure implements what domain and application define. Shell calls application and owns no logic.

Each domain component (COMP-1, COMP-2, COMP-3) has its own `domain/`, `application/`, and `infrastructure/` layers. COMP-4 (`shell`) is the system entry point and composition root — it has no DDD layers.

## COMP-1 — `conversation_management`

### Domain Layer — `src/conversation_management/domain/`

Owns the pure domain model for conversation lifecycle. Enforces all INV-CS-* and INV-WF-* invariants. No framework. No I/O.

| Type | Name | Purpose |
|------|------|---------|
| Process Entity | `ConversationSession` | Lifecycle owner. State machine: created → initializing → idle ⇄ active → closed. Enforces all session state and turn sequencing invariants. |
| Entity | `ConversationHistory` | Persisted turn records for a user. Loaded at session init to restore prior context. Immutable historical entity — append-only log. |
| Value Object | `User` | External reference value object. Identity reference pre-resolved by external auth — never authenticated here. Immutable reference to User entity in external Auth BC. |
| Value Object | `School` | External reference value object. School the user supports. Loaded from DB at init. Anchors the fundraising context pre-seeded into DecisionContext. Immutable reference to School entity in external Master Data BC. |
| Value Object | `Intent` | Classified purpose of a turn. Immutable. Values: `product_inquiry`, `referral_question`, `general_question`, `clarification_response`, `ambiguous`, `out_of_scope`. |
| Value Object | `AssistantResponse` | Grounded reply returned to the caller. Carries `groundedness_score` and `relevance_score`. Must clear score thresholds before delivery. |
| Domain Event | `SessionStarted` | Emitted when session becomes active (first turn ready). |
| Domain Event | `TurnProcessed` | Emitted when session returns to idle (turn completed). |
| Domain Event | `SessionClosed` | Emitted when session is closed. |
| Repository Interface | `IConversationSessionRepository` | Persistence contract for ConversationSession. |
| Repository Interface | `IConversationHistoryRepository` | Read contract for loading conversation history at session init. |
| Repository Interface | `IUserRepository` | Read contract for User identity by user_id. |
| Repository Interface | `ISchoolRepository` | Read contract for School associated with a user. |
| Repository Interface | `IPurchaseHistoryRepository` | Read contract for PurchaseHistory. Optional signal — skipped if empty. |

---

### Application Layer — `src/conversation_management/application/`

Orchestrates the conversation turn loop. Calls domain transitions, external ports, and Decision Intelligence. **LangGraph is the execution engine for the turn loop** — nodes are thin wrappers that call ports; domain logic stays in the domain layer.

| Type | Name | Purpose |
|------|------|---------|
| Use Case | `StartSessionUseCase` | Initializes session: loads User, School, ConversationHistory, PurchaseHistory; pre-seeds DecisionContext via `IDecisionIntelligenceService`; transitions session to idle. |
| Use Case | `ProcessTurnUseCase` | Entry point for a single conversation turn. Delegates execution to `TurnGraph`. Returns validated `AssistantResponse` to caller. |
| Use Case | `CloseSessionUseCase` | Transitions session to closed. Final persistence. |
| LangGraph Graph | `TurnGraph` | Executes the full turn pipeline. Replaces the imperative body of `ProcessTurnUseCase`. Nodes call ports — no domain logic inside nodes. Conditional edges: `classify_intent` routes by intent value; `evaluate_response` branches on score thresholds. |
| LangGraph State | `TurnGraphState` | Typed state schema for `TurnGraph`. Turn-scoped primitives only — no domain objects. Fields: `user_id`, `session_id`, `message`, `intent`, `evidence_chunks`, `evidence_type`, `response_text`, `groundedness_score`, `relevance_score`, `clarification_needed`. |
| Port | `IIntentClassifier` | Contract for classifying Intent from a user message. Implemented by LLM adapter. |
| Port | `IResponseGenerator` | Contract for generating AssistantResponse grounded in RetrievedEvidence + context snapshot. Implemented by LLM adapter. |
| Port | `IResponseEvaluator` | Contract for computing `groundedness_score` and `relevance_score`. Enforces INV-AR-1, INV-AR-5, INV-AR-6. |
| Port | `IDecisionIntelligenceService` | Cross-component port to COMP-2. Delivers intent + message; receives RetrievedEvidence. No COMP-2 domain objects cross into COMP-1 domain. |
| Port | `IRAGMetricsLogger` | Observability port. Persists per-turn RAG evaluation metrics (groundedness, relevance, context scores, fallback) to rag_metrics.sqlite. |

---

### Infrastructure Layer — `src/conversation_management/infrastructure/`

Implements all ports and repository interfaces. All I/O lives here.

| Type | Name | Implements | Purpose |
|------|------|-----------|---------|
| Repository | `InMemoryConversationSessionRepository` | `IConversationSessionRepository` | In-memory dict store for active session state. Session-scoped. |
| Repository | `SQLiteConversationHistoryRepository` | `IConversationHistoryRepository` | SQLite adapter for Conversation_History table. Primary history store. |
| Repository | `InMemoryConversationHistoryRepository` | `IConversationHistoryRepository` | In-memory variant — dev/test only. |
| Repository | `SQLiteUserRepository` | `IUserRepository` | SQLite adapter for User identity reads. |
| Repository | `SQLiteSchoolRepository` | `ISchoolRepository` | SQLite adapter for School reads by user association. |
| Repository | `SQLitePurchaseHistoryRepository` | `IPurchaseHistoryRepository` | SQLite adapter for Purchase_History table. Returns empty gracefully. |
| Adapter | `OpenAIIntentClassifier` | `IIntentClassifier` | LLM adapter (gpt-4o-mini). Classifies user message into one of the six Intent values. |
| Adapter | `OpenAIResponseGenerator` | `IResponseGenerator` | LLM adapter (gpt-4o). Generates grounded AssistantResponse from evidence + context. |
| Adapter | `OpenAIResponseEvaluator` | `IResponseEvaluator` | Scoring adapter (gpt-4o-mini). Computes `groundedness_score` and `relevance_score`. Blocks response if either score is below threshold. |
| Adapter | `SQLiteRAGMetricsLogger` | `IRAGMetricsLogger` | Persists per-turn RAG evaluation metrics to rag_metrics.sqlite. |

## COMP-2 — `decision_intelligence`

### Domain Layer — `src/decision_intelligence/domain/`

Owns the domain model for preference accumulation and retrieval query construction. Enforces INV-DC-* and INV-RQ-* invariants. No framework. No I/O.

| Type | Name | Purpose |
|------|------|---------|
| Process Entity | `DecisionContext` | Accumulates PreferenceSignals monotonically. State: empty → partial → ready → enriched. Context never regresses. School always present. Signals never removed. Enforces INV-DC-1 through INV-DC-6. |
| Entity | `RetrievalQuery` | Context-enriched query artifact. Built from Intent + DecisionContext. Carries `source_target`. Never constructed from raw message. Enforces INV-RQ-1, INV-RQ-2, INV-RQ-3. |
| Value Object | `PreferenceSignal` | Single preference/constraint from one turn. Immutable. Examples: `kids=picky`, `budget=sensitive`. Accumulates monotonically. |
| Value Object | `RetrievedEvidence` | Retrieved content returned from COMP-3. Immutable. Carries `context_recall_score` and `context_precision_score`. Quality-gated before passing to COMP-1. |
| Domain Event | `ContextUpdated` | Emitted when a PreferenceSignal is added and state changes. |
| Domain Event | `GapDetected` | Emitted when gap evaluation finds context insufficient — triggers clarification in COMP-1. |
| Domain Event | `ContextReady` | Emitted when context transitions to ready for the first time. |
| Repository Interface | `IDecisionContextRepository` | Persistence contract for DecisionContext per session. |

---

### Application Layer — `src/decision_intelligence/application/`

Orchestrates signal extraction, context evaluation, query construction, retrieval execution, and evidence quality evaluation for each turn. **LangGraph is the execution engine** — the `DecisionIntelligenceSubgraph` replaces the imperative orchestration that previously lived in `ProcessTurnIntelligenceUseCase`.

| Type | Name | Purpose |
|------|------|---------|
| Use Case | `PreSeedContextUseCase` | Called at session init by COMP-1. Creates DecisionContext with school anchor and any available PurchaseHistory signals. |
| Use Case | `ProcessTurnIntelligenceUseCase` | Entry point for COMP-2 per turn. Delegates execution to `DecisionIntelligenceSubgraph`. Returns `RetrievedEvidence` (or clarification signal) to COMP-1. |
| LangGraph Graph | `DecisionIntelligenceSubgraph` | Executes the full decision intelligence pipeline per turn. Nodes: `extract_signals`, `update_decision_context`, `evaluate_context_readiness`, `construct_retrieval_query`, `execute_retrieval`, `evaluate_evidence_quality`. Conditional edges: gap detection branches to clarification; low recall score cycles back to retrieval with refined query. |
| LangGraph State | `DecisionGraphState` | Typed state schema for `DecisionIntelligenceSubgraph`. Turn-scoped primitives only. Fields: `session_id`, `intent`, `message`, `signals`, `context_state`, `retrieval_query_text`, `source_target`, `evidence_chunks`, `recall_score`, `precision_score`, `clarification_needed`. |
| Port | `ISignalExtractor` | Contract for extracting PreferenceSignals from a message using LLM. |
| Port | `IEvidenceEvaluator` | Contract for computing `context_recall_score` and `context_precision_score`. If recall < threshold, signals query refinement before injection. |
| Port | `IKnowledgeRetrievalService` | Cross-component port to COMP-3. Sends RetrievalQuery; receives RetrievedEvidence. |

---

### Infrastructure Layer — `src/decision_intelligence/infrastructure/`

| Type | Name | Implements | Purpose |
|------|------|-----------|---------|
| Repository | `InMemoryDecisionContextRepository` | `IDecisionContextRepository` | In-memory dict store for DecisionContext. Scoped to session lifetime. |
| Adapter | `OpenAISignalExtractor` | `ISignalExtractor` | LLM adapter (gpt-4o-mini). Extracts PreferenceSignal values from a user message. |
| Adapter | `OpenAIEvidenceEvaluator` | `IEvidenceEvaluator` | Scoring adapter (gpt-4o-mini). Computes `context_recall_score` and `context_precision_score` on RetrievedEvidence before injection into generation. |

## COMP-3 — `knowledge_retrieval`

### Domain Layer — `src/knowledge_retrieval/domain/`

Owns the two knowledge sources. Defines semantic retrieval contracts. Enforces the invariant that content is only retrieved — never invented or rewritten.

| Type | Name | Purpose |
|------|------|---------|
| Entity | `Product` | Catalog item. Retrieved by semantic relevance. Never invented. Carries identity, name, category, brand, and attributes. |
| Entity | `ReferralRule` | Platform-defined referral rule. Immutable. Retrieved by semantic relevance. Never rewritten. |
| Value Object | `RetrievedEvidence` | Retrieval result. Typed as `product_evidence` or `referral_rule_evidence`. Immutable. |
| Repository Interface | `IProductRepository` | Semantic retrieval contract for product_catalog. |
| Repository Interface | `IReferralRuleRepository` | Semantic retrieval contract for referral_program_rules. |

---

### Application Layer — `src/knowledge_retrieval/application/`

| Type | Name | Purpose |
|------|------|---------|
| Use Case | `ExecuteRetrievalUseCase` | Receives RetrievalQuery, routes to the correct repository by `source_target` (`product_catalog` → `IProductRepository`; `referral_program_rules` → `IReferralRuleRepository`). Returns RetrievedEvidence to COMP-2. |

---

### Infrastructure Layer — `src/knowledge_retrieval/infrastructure/`

| Type | Name | Implements | Purpose |
|------|------|-----------|---------|
| Adapter | `QdrantProductRepository` | `IProductRepository` | Qdrant vector store adapter for semantic search against product_catalog collection. Returns top-matching product evidence. |
| Adapter | `QdrantReferralRuleRepository` | `IReferralRuleRepository` | Qdrant vector store adapter for semantic search against referral_program_rules collection. Returns top-matching rule evidence. |

## COMP-4 — `shell`

Not a domain component. No DDD layers. System entry point and composition root.

### API Layer — `src/shell/api/`

| Type | Name | Purpose |
|------|------|---------|
| Schemas | `StartSessionRequest`, `ProcessTurnRequest`, `CloseSessionRequest` | Inbound Pydantic schemas. `StartSessionRequest`: `user_id`. `ProcessTurnRequest`: `user_id`, `message`. `CloseSessionRequest`: `user_id`. |
| Schemas | `StartSessionResponse`, `ProcessTurnResponse`, `CloseSessionResponse`, `ErrorResponse` | Outbound Pydantic schemas. `ProcessTurnResponse` carries: `session_id`, `response_text`, `intent`, `is_fallback`, `groundedness_score`, `context_precision_score`, `context_recall_score`, `relevance_score`. |
| Router | `build_router()` | Factory function that creates the FastAPI `APIRouter` with three endpoints: `POST /sessions`, `POST /sessions/{session_id}/turns`, `DELETE /sessions/{session_id}`. |
| App Factory | `create_app()` | FastAPI application factory. Calls `build_container()`, instantiates the router, and registers it on the app. Returns a fully configured `FastAPI` instance. |

### Composition Layer — `src/shell/composition/`

| Type | Name | Purpose |
|------|------|---------|
| Composition Root | `build_container()` | Factory function in `container.py`. The only place that imports from all components simultaneously. Instantiates repositories and adapters, injects them into use cases, binds ports to implementations, and returns a fully wired `AppContainer`. |

## Cross-Component Communication

| From | To | Mechanism | Payload |
|------|----|-----------|---------|
| COMP-1 application | COMP-2 application | `IDecisionIntelligenceService` port | Intent + message → RetrievedEvidence |
| COMP-2 application | COMP-3 application | `IKnowledgeRetrievalService` port | RetrievalQuery → RetrievedEvidence |

**Key rule:** Cross-component calls cross at the **application layer** only — via defined port interfaces. No domain object from one component is imported into another component's domain layer. COMP-4 `Container` binds the concrete implementations to these ports at startup.

## RAG Pipeline Mapped to Layers

| Pipeline Step | Component | Layer | Artifact |
|--------------|-----------|-------|----------|
| Receive message | COMP-1 | application | `ProcessTurnUseCase` |
| Classify intent | COMP-1 | infra → application | `OpenAIIntentClassifier` via `IIntentClassifier` |
| Extract signals | COMP-2 | infra → application | `OpenAISignalExtractor` via `ISignalExtractor` |
| Accumulate context | COMP-2 | domain | `DecisionContext` |
| Build retrieval query | COMP-2 | application | `ProcessTurnIntelligenceUseCase` → `RetrievalQuery` |
| Execute retrieval | COMP-3 | infra → application | `QdrantProductRepository` / `QdrantReferralRuleRepository` |
| Evaluate evidence quality | COMP-2 | infra → application | `OpenAIEvidenceEvaluator` via `IEvidenceEvaluator` |
| Generate response | COMP-1 | infra → application | `OpenAIResponseGenerator` via `IResponseGenerator` |
| Evaluate response quality | COMP-1 | infra → application | `OpenAIResponseEvaluator` via `IResponseEvaluator` |
| Return validated response | COMP-1 → shell | application → api | `ProcessTurnResponse` |
