**Task:** Use Case Generator

---

## Design Principles

- Each use case is **one atomic system action** — one transaction boundary.
- Use cases **orchestrate domain behavior** — they never implement domain rules or invariants directly; the domain handles that.
- Flow per use case: load → invoke domain → persist → drain events → return.
- **Cross-context calls** go exclusively through ports defined in the caller's application layer; adapters live only in the shell.
- No domain object from one bounded context is passed into another — **only primitive DTOs cross boundaries**.
- LangGraph graphs (`TurnGraph`, `DecisionIntelligenceSubgraph`) are the execution engines wired *inside* their parent use case. The use case is the entry point and transaction owner; graph node/edge definitions are materialized in Task 2 (Workflow Orchestration).

## BC-1 — Conversation Management

### UC-1: `StartSessionUseCase`

**Module:** `src/conversation_management/application/use_cases/start_session_use_case.py`  
**Workflow:** WF0 — session initialization (pre-turn)

**Purpose:** Creates and fully initializes a `ConversationSession`. Enforces the one-session-per-user constraint. Loads all identity and history data from the DB. Pre-seeds `DecisionContext` in BC-2. Transitions the session to `idle` state, ready for the first turn.

**Input Command:**
```
StartSessionCommand
  user_id: str   # Pre-resolved trusted identity — never authenticated here
```

**Output Response:**
```
StartSessionResponse
  session_id: str
  status: str    # "idle"
```

**Dependencies:**
- `IUserRepository` — load `User` by `user_id`
- `ISchoolRepository` — load `School` by `user_id`
- `IConversationHistoryRepository` — load prior history by `user_id`
- `IPurchaseHistoryRepository` — load purchase signals (empty list if none)
- `IConversationSessionRepository` — find existing active/idle session; save new session
- `IDecisionIntelligenceService` — `pre_seed_context(PreSeedContextDTO)` → COMP-2

**Execution Flow:**
1. Validate `user_id` is not empty
2. `IConversationSessionRepository.find_active_by_user_id(user_id)` — raise `ConflictError` if active/idle session exists **(INV-CS-7)**
3. `IUserRepository.find_by_id(user_id)` — raise `NotFoundError` if user not found **(INV-CS-6)**
4. `ISchoolRepository.find_by_user_id(user_id)` — raise `NotFoundError` if school not found **(INV-CS-3)**
5. `IConversationHistoryRepository.find_by_user_id(user_id)` — returns `[]` gracefully
6. `IPurchaseHistoryRepository.find_by_user_id(user_id)` — returns `[]` gracefully
7. `ConversationSession.create(user_id, school_id)` — session enters `created` state
8. `session.begin_initialization()` — transitions `created → initializing`
9. Build `PreSeedContextDTO(session_id, school_id, school_name, purchase_signals)`
10. `IDecisionIntelligenceService.pre_seed_context(dto)` → delegates to **UC-4** **(INV-CS-3, INV-CS-5)**
11. `session.complete_initialization()` — transitions `initializing → idle`; emits `SessionStarted`
12. `IConversationSessionRepository.save(session)`
13. Drain domain events (`SessionStarted`)
14. Return `StartSessionResponse`

**Invariants Enforced:** INV-CS-3, INV-CS-5, INV-CS-6, INV-CS-7

### UC-2: `ProcessTurnUseCase`

**Module:** `src/conversation_management/application/use_cases/process_turn_use_case.py`  
**Workflow:** WF0 — per-turn execution loop

**Purpose:** Entry point for every conversation turn. Guards session state. Runs `TurnGraph` (LangGraph). The graph classifies intent, delegates to Decision Intelligence, generates a response, evaluates quality, and returns either a grounded `AssistantResponse` or a safe fallback. Persists the completed turn.

**Input Command:**
```
ProcessTurnCommand
  session_id: str
  user_id:    str   # For history persistence
  message:    str   # Raw user message
```

**Output Response:**
```
ProcessTurnResponse
  session_id:              str
  response_text:           str
  intent:                  str
  is_fallback:             bool
  groundedness_score:      float
  context_precision_score: float
  context_recall_score:    float
  relevance_score:         float
```

**Dependencies:**
- `IConversationSessionRepository` — load session, save after turn
- `IConversationHistoryRepository` — append user + assistant history entries
- `IUserRepository` — load `User.name` for LLM personalization
- `ISchoolRepository` — load `School.name` for LLM personalization
- `IIntentClassifier` — classify `Intent` from message *(TurnGraph node)*
- `IDecisionIntelligenceService` — `process_turn(TurnInputDTO)` → `TurnIntelligenceResultDTO` *(TurnGraph node)*
- `IResponseGenerator` — generate grounded `AssistantResponse` *(TurnGraph node)*
- `IResponseEvaluator` — score groundedness and relevance *(TurnGraph node)*
- `IRAGMetricsLogger` — persist per-turn RAG evaluation metrics after TurnGraph completes

**Execution Flow:**
1. `IConversationSessionRepository.find_by_id(session_id)` — raise `NotFoundError` if not found
2. Guard: session must be in `idle` state — raise `InvalidStateError` if `initializing`, `created`, or `closed` **(INV-CS-4)**
3. Guard: `session.user_id == command.user_id` — raise `SessionOwnershipError` if mismatch **(INV-CS-5)**
4. Load full `ConversationHistory` via `find_by_user_id(user_id)`. Map to `conversation_history: list[dict]` with `{"role": "user"|"assistant", "content": str}` pairs (Proposal B — explicit history injection)
4a. `IUserRepository.find_by_id(user_id)` — load `User.name` as `user_name` (default `""` if not found)
4b. `ISchoolRepository.find_by_user_id(user_id)` — load `School.name` as `school_name` (default `""` if not found)
    Both values are injected into `TurnGraphState` so all LLM nodes (N3, N4, N8) can personalise their system prompts via `OpenAIResponseGenerator`
5. `session.receive_message()` — transitions `idle → active`
6. **Open try block covering steps 7–12:**
7. Build initial `TurnGraphState(session_id, user_id, message, turn_number=session.turn_count+1)`
8. **Run TurnGraph** *(full node/edge spec in Task 2 — Workflow Orchestration)*:
   - `classify_intent` node → `IIntentClassifier.classify(message)` → `Intent`
   - Conditional routing on intent:
     - `out_of_scope` → `handle_out_of_scope` node (polite scope-redirect; DI and retrieval bypassed; history not persisted)
     - `ambiguous` → `generate_clarification` node
     - `general_question` → `generate_redirect` node (redirect; DI and retrieval bypassed)
     - `product_inquiry` / `referral_question` / `clarification_response` → `process_decision_intelligence` node
   - `process_decision_intelligence` node → `IDecisionIntelligenceService.process_turn(TurnInputDTO)` → `TurnIntelligenceResultDTO`
     - If `clarification_needed=True` → route to `generate_clarification` node
   - `generate_response` node → `IResponseGenerator.generate(message, evidence, context_snapshot)` → `AssistantResponse`
   - `evaluate_response` node → `IResponseEvaluator.evaluate(response, intent, evidence)` → scored response
     - Scores pass → `persist_turn` node
     - Scores fail → `return_fallback` node → `AssistantResponse.fallback()`
9. `session.complete_turn(intent, response)` — transitions `active → idle`; emits `TurnProcessed`; increments `turn_count`
10. **On any exception in steps 7–9:** call `session.fail_turn()` — transitions `active → idle` (rollback, no event emitted); call `IConversationSessionRepository.save(session)`; re-raise so the API layer returns the correct HTTP error
11. Append `ConversationHistory` entries (user message + assistant response)
12. `IConversationSessionRepository.save(session)`
13. `IConversationHistoryRepository.append()` for both entries
14. Drain domain events (`TurnProcessed`)
15. Return `ProcessTurnResponse`

**Invariants Enforced:** INV-CS-4, INV-WF-1, INV-AR-1, INV-AR-5, INV-AR-6

**Cross-Context Call:**
```
IDecisionIntelligenceService.process_turn(TurnInputDTO) → TurnIntelligenceResultDTO
  delegates to: ProcessTurnIntelligenceUseCase (COMP-2) via DecisionIntelligenceAdapter (shell)
```

### UC-3: `CloseSessionUseCase`

**Module:** `src/conversation_management/application/use_cases/close_session_use_case.py`  
**Workflow:** WF0 — session closure

**Purpose:** Transitions a `ConversationSession` to the `closed` terminal state. Enforces that only `idle` sessions can be closed. Persists the final state.

**Input Command:**
```
CloseSessionCommand
  session_id: str
  user_id:    str   # Ownership verification
```

**Output Response:**
```
CloseSessionResponse
  session_id: str
  status: str    # "closed"
```

**Dependencies:**
- `IConversationSessionRepository` — load session, save terminal state

**Execution Flow:**
1. `IConversationSessionRepository.find_by_id(session_id)` — raise `NotFoundError` if not found
2. Guard: `session.user_id == command.user_id` — raise `AuthorizationError` if mismatch
3. `session.close()` — domain guard enforces `status == idle`; transitions `idle → closed`; emits `SessionClosed` *(raises `DomainError` if session is `active` — enforces INV-CS-2)*
4. `IConversationSessionRepository.save(session)`
5. Drain domain events (`SessionClosed`)
6. Return `CloseSessionResponse`

**Invariants Enforced:** INV-CS-2 (`active → closed` is illegal; domain raises `DomainError`)

## BC-2 — Decision Intelligence

### UC-4: `PreSeedContextUseCase`

**Module:** `src/decision_intelligence/application/use_cases/pre_seed_context_use_case.py`  
**Workflow:** WF0 — session init (called via COMM-1)

**Purpose:** Creates a `DecisionContext` pre-seeded with the school anchor and any available purchase signals. Called once per session during initialization — before any turn.

**Input Command:**
```
PreSeedContextCommand
  session_id:       str
  school_id:        str
  school_name:      str
  purchase_signals: list[str]   # Raw signal strings — empty list if no purchase history
```

**Output Response:** `None` (void — called for its side effect)

**Dependencies:**
- `IDecisionContextRepository` — save newly created `DecisionContext`

**Execution Flow:**
1. Build `SchoolAnchor(school_id, school_name)` value object
2. `DecisionContext.create(session_id, school_anchor)` — context enters `empty` state with school pre-seeded **(INV-DC-2)**
3. If `purchase_signals` is non-empty: for each signal, create `PreferenceSignal(key='purchase_history', value=signal, turn_number=0)` and call `context.add_signal()` — may transition `empty → partial`
4. `IDecisionContextRepository.save(context)`
5. Return `None`

**Invariants Enforced:** INV-DC-2, INV-CS-3

**Note:** Called indirectly by COMP-1 via `IDecisionIntelligenceService.pre_seed_context()` → `DecisionIntelligenceAdapter` (shell) → this use case. COMP-1 never imports this use case directly.

### UC-5: `ProcessTurnIntelligenceUseCase`

**Module:** `src/decision_intelligence/application/use_cases/process_turn_intelligence_use_case.py`  
**Workflow:** WF1 + WF2 + WF3 — per turn

**Purpose:** Entry point for COMP-2 per conversation turn. Runs `DecisionIntelligenceSubgraph` (LangGraph). The subgraph extracts preference signals, updates `DecisionContext`, evaluates readiness, constructs the retrieval query, executes retrieval via COMP-3, and evaluates evidence quality. Returns a `TurnIntelligenceResult` (with clarification signal if gaps are detected).

**Input Command:**
```
ProcessTurnIntelligenceCommand
  session_id:           str
  intent:               str   # Intent value as string — not BC-1 domain object
  message:              str   # Raw user message
  turn_number:          int   # From ConversationSession.turn_count — used for PreferenceSignal timestamping
  conversation_history: list[dict] = []   # Windowed prior conversation context in OpenAI message format
```

**Output Response:**
```
TurnIntelligenceResult
  evidence_chunks:          list[str]   # Empty if clarification needed
  evidence_type:            str         # "product_evidence" | "referral_rule_evidence" | "none"
  context_recall_score:     float       # 0.0 if no retrieval occurred
  context_precision_score:  float       # 0.0 if no retrieval occurred
  context_state:            str         # "empty" | "partial" | "ready" | "enriched"
  clarification_needed:     bool
  clarification_question:   str | None
  school_name:              str = ""    # Forwarded from SchoolAnchor for LLM personalization in BC-1
```

**Dependencies:**
- `IDecisionContextRepository` — load `DecisionContext` by `session_id`, save after update
- `ISignalExtractor` — extract `PreferenceSignal` list from message *(subgraph node)*
- `IEvidenceEvaluator` — compute recall/precision scores on evidence *(subgraph node)*
- `IKnowledgeRetrievalService` — retrieve evidence from COMP-3 *(subgraph node)*

**Execution Flow:**
1. `IDecisionContextRepository.find_by_session_id(session_id)` — raise `NotFoundError` if not found
2. Build initial `DecisionGraphState(session_id, intent, message, turn_number, context_snapshot=context.snapshot())`
3. **Run DecisionIntelligenceSubgraph** *(full node/edge spec in Task 2 — Workflow Orchestration)*:
   - `extract_signals` node → `ISignalExtractor.extract(message, turn_number)` → `list[PreferenceSignal]`
   - `update_decision_context` node → `context.add_signal()` for each signal; context state transitions as warranted
   - `evaluate_context_readiness` node → `context.record_gap_evaluation(gaps)` → `context.mark_ready()` if sufficient
   - Conditional routing on readiness and intent:
     - Gap detected AND intent doesn't require retrieval → `clarification_needed=True`, skip retrieval
     - `intent.requires_retrieval()` AND `context.is_ready_for_retrieval()` → `construct_retrieval_query` node **(INV-DC-4)**
     - `general_question` / `clarification_response` intent → skip retrieval entirely
   - `construct_retrieval_query` node → `RetrievalQuery.from_context(session_id, enriched_text, intent, context_status)` — query built from `context.snapshot()`, not raw message **(INV-RQ-1, INV-RQ-2, INV-RQ-3)**
   - `execute_retrieval` node → `IKnowledgeRetrievalService.retrieve(RetrievalQueryDTO)` → `RetrievedEvidenceDTO`
   - `evaluate_evidence_quality` node → `IEvidenceEvaluator.evaluate(query, evidence)` → scored evidence
     - Recall < threshold → refine query and retry `execute_retrieval` **(INV-RE-3)**
     - Quality passes → return evidence **(INV-RE-4)**
4. `IDecisionContextRepository.save(context)` — persist updated signals and state
5. Drain domain events (`ContextUpdated`, `GapDetected`, `ContextReady` — intra-component only)
6. Return `TurnIntelligenceResult`

**Invariants Enforced:** INV-DC-4, INV-DC-5, INV-RQ-1, INV-RQ-2, INV-RQ-3, INV-RE-1, INV-RE-2, INV-RE-3, INV-RE-4, INV-WF-2

**Cross-Context Call:**
```
IKnowledgeRetrievalService.retrieve(RetrievalQueryDTO) → RetrievedEvidenceDTO
  delegates to: ExecuteRetrievalUseCase (COMP-3) via KnowledgeRetrievalAdapter (shell)
```

## BC-3 — Knowledge Retrieval

### UC-6: `ExecuteRetrievalUseCase`

**Module:** `src/knowledge_retrieval/application/use_cases/execute_retrieval_use_case.py`  
**Workflow:** WF2 (Product Recommendation) + WF3 (Referral Guidance)

**Purpose:** Routes a retrieval query to the correct knowledge store based on `source_target`. Returns raw retrieved content as `RetrievedEvidenceDTO`. Never invents or rewrites content — only returns what the knowledge store provides.

**Input Command:**
```
ExecuteRetrievalCommand
  query_text:    str   # Enriched query text — constructed by COMP-2 from context snapshot
  source_target: str   # "product_catalog" | "referral_program_rules"
```

**Output Response:**
```
RetrievedEvidenceDTO                 (defined in src/decision_intelligence/application/ports/)
  chunks:         list[str]          # Top-matching content text from vector store
  evidence_type:  str                # "product_evidence" | "referral_rule_evidence"
```

**Dependencies:**
- `IProductRepository` — semantic search over `product_catalog` collection
- `IReferralRuleRepository` — semantic search over `referral_program_rules` collection

**Execution Flow:**
1. Validate `source_target` is `product_catalog` or `referral_program_rules` — raise `InvalidTargetError` otherwise
2. Route on `source_target`:
   - `product_catalog` → `IProductRepository.search(query_text)` → `list[Product]` **(INV-AR-2)**
   - `referral_program_rules` → `IReferralRuleRepository.search(query_text)` → `list[ReferralRule]` **(INV-AR-3)**
3. Map retrieved entities to `RetrievedEvidenceDTO`:
   - `chunks`: list of content text strings extracted from retrieved entities
   - `evidence_type`: `product_evidence` or `referral_rule_evidence`
4. Return `RetrievedEvidenceDTO`

**Invariants Enforced:** INV-AR-2, INV-AR-3, INV-AR-4

**Note:** This is the leaf use case in the system dependency chain — COMP-3 has no outbound dependencies. Quality evaluation (recall/precision scoring) happens in COMP-2 after this use case returns. Invoked by `KnowledgeRetrievalAdapter` (shell) on behalf of COMP-2.

## System Call Chain

### Session Initialization
```
shell/api → StartSessionUseCase [UC-1, BC-1]
  └─→ IDecisionIntelligenceService.pre_seed_context → DecisionIntelligenceAdapter (shell)
        └─→ PreSeedContextUseCase [UC-4, BC-2]
  ← StartSessionResponse
```

### Per-Turn Execution
```
shell/api → ProcessTurnUseCase [UC-2, BC-1]
  └─→ TurnGraph
        ├─ IIntentClassifier.classify(message)
        └─→ IDecisionIntelligenceService.process_turn → DecisionIntelligenceAdapter (shell)
              └─→ ProcessTurnIntelligenceUseCase [UC-5, BC-2]
                    └─→ DecisionIntelligenceSubgraph
                          └─→ IKnowledgeRetrievalService.retrieve → KnowledgeRetrievalAdapter (shell)
                                └─→ ExecuteRetrievalUseCase [UC-6, BC-3]
                                ← RetrievedEvidenceDTO
                          └─ IEvidenceEvaluator.evaluate
                    ← TurnIntelligenceResult
              ← TurnIntelligenceResultDTO
        └─ IResponseGenerator.generate
        └─ IResponseEvaluator.evaluate
  ← ProcessTurnResponse
```

### Session Closure
```
shell/api → CloseSessionUseCase [UC-3, BC-1]
  ← CloseSessionResponse
```

## Cross-Context DTOs

All DTOs are plain data—no domain objects. Defined adjacent to the port they serve.

| DTO                       | Location                                     | Direction         | Carries                                                        |
|---------------------------|----------------------------------------------|-------------------|----------------------------------------------------------------|
| `PreSeedContextDTO`       | `conversation_management/application/ports/` | COMP-1 → COMP-2   | `session_id`, `school_id`, `school_name`, `purchase_signals`  |
| `TurnInputDTO`            | `conversation_management/application/ports/` | COMP-1 → COMP-2   | `session_id`, `intent`, `message`, `turn_number`              |

| `RetrievedEvidenceDTO`    | `decision_intelligence/application/ports/`   | COMP-3 → COMP-2   | `chunks`, `evidence_type`                                      |
